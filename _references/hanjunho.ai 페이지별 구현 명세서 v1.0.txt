hanjunho.ai 페이지별 구현 명세서 v1.0
부제: 웹플로우 디자이너·개발자용 페이지 단위 구현 기준서
1. 문서 목적
이 문서는 hanjunho.ai의 각 페이지를 무슨 역할로 설계할지, 어떤 블록 순서로 구현할지, 어떤 CMS 필드를 어디에 연결할지, 무엇을 삭제하고 무엇을 추가할지를 페이지 단위로 명확히 규정하기 위한 문서다.
이 문서의 목적은 세 가지다.
첫째, 디자이너가 페이지마다 다른 시각적 위계와 정보 강조점을 정확히 이해하게 한다.둘째, 개발자가 Webflow에서 페이지, 컬렉션 리스트, 템플릿, 필터, 메타 연결을 혼동 없이 구현하게 한다.셋째, 운영자가 향후 카드와 브리핑을 추가할 때 어떤 페이지가 어떤 콘텐츠를 담아야 하는지 일관되게 유지하게 한다.
이 문서는 IA·사이트맵 명세서와 CMS 컬렉션·필드 사전을 전제로 읽어야 한다.즉, 이 문서는 “무슨 페이지가 있어야 하는가”가 아니라 **“그 페이지를 실제로 어떻게 구성해야 하는가”**에 대한 기준 문서다.
2. 공통 구현 원칙
2.1 모든 페이지는 역할이 분명해야 한다
각 페이지는 하나의 핵심 목적만 가져야 한다.소개, 탐색, 정답, 해설, 검증, 신뢰, 행동 전환이 한 페이지 안에서 뒤섞이면 안 된다.
2.2 페이지 상단 1화면 안에서 정체성이 드러나야 한다
사용자는 스크롤하기 전에 다음을 이해해야 한다.
이 페이지가 무엇을 위한 페이지인지누가 말하는지무슨 행동을 할 수 있는지
2.3 상위 허브와 정본 상세를 분리한다
홈과 클리어맵, 이슈브리핑 인덱스는 허브다.Answer Card 상세와 이슈브리핑 상세는 정본이다.허브는 요약과 진입을 담당하고, 정본은 설명과 신뢰를 담당한다.
2.4 상세 페이지는 “첫 화면 핵심 정보”가 중요하다
Answer Card 상세는질문, 3문장 답, 검수, 업데이트일이 첫 화면에서 보여야 한다.
이슈브리핑 상세는기사 출처, 요약, 핵심 쟁점이 첫 화면에서 보여야 한다.
2.5 삭제 작업도 명세의 일부다
현재 사이트에 남아 있는 기업형 템플릿 카피, 범용 브리핑, 샘플 카드, 잘못된 검수자 표기, GNB 혼선 요소는 삭제 또는 비공개가 구현 항목에 포함된다.무엇을 넣을지 못지않게 무엇을 빼야 하는지가 중요하다.

3.1 전역 헤더(GNB)

현재 상단에 섞여 있는 5개 pillar 링크, Gallery, Contact는 GNB에서 제거한다.
구현 지침

3.2 전역 푸터

19. 최종 요약
이 문서의 핵심은 다음과 같다.
각 페이지는 하나의 역할만 가져야 한다.허브 페이지는 진입과 탐색을 담당하고, 상세 페이지는 정본 역할을 담당해야 한다.Ask AI는 질문 진입 허브,클리어맵은 정책 탐색 허브,Answer Card는 정책 정본,이슈브리핑은 기사형 해설 정본,팩트 인증센터는 신뢰 허브,한준호와 김태영 교수 페이지는 각각 공식 발화자와 총괄 검수자 권위 페이지가 되어야 한다.
이 페이지별 구현 명세서는 이후카피 마스터 문서,AEO·SEO·Schema 기술 명세서,QA/UAT 체크리스트작성의 직접 기준으로 사용한다.
