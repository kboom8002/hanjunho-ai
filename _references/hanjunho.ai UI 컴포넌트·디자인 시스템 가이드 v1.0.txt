hanjunho.ai UI 컴포넌트·디자인 시스템 가이드 v1.0
부제: 웹플로우 디자이너·프론트 구현자를 위한 시각 시스템, 컴포넌트 규칙, 반응형 기준서
1. 문서 목적
이 문서는 hanjunho.ai의 전반적인 시각 언어와 UI 컴포넌트 규칙을 일관되게 정리하기 위한 기준 문서다.목적은 세 가지다.
첫째, 디자이너가 페이지마다 다른 스타일을 임의로 만들지 않고, 하나의 통일된 디자인 시스템 안에서 작업하도록 한다.둘째, 개발자가 Webflow에서 컴포넌트와 클래스 구조를 일관되게 구현할 수 있도록 기준을 제공한다.셋째, hanjunho.ai를 단순 선거용 홍보 사이트가 아니라 질문형 정책 플랫폼, 기사형 이슈 해설 허브, 신뢰 인증 허브로 인식하게 만드는 시각적 정체성을 확립한다.
이 문서는 단순한 “색상표”가 아니다.이 문서는 정치 전략, 정보구조, UX, AEO 관점이 반영된 시각 운영 기준서다.
2. 디자인 시스템의 최상위 개념
2.1 한 줄 정의
hanjunho.ai의 UI는“캠프 홈페이지”보다 “공식 질문형 정책 플랫폼”처럼 보여야 한다.
즉 사용자가 사이트에 들어왔을 때,“홍보물”을 보는 느낌보다“질문에 답하는 신뢰 가능한 공공 플랫폼”을 보는 느낌이 먼저 와야 한다.
2.2 시각적 방향성
시각적 방향성은 다음 네 가지를 동시에 만족해야 한다.
첫째, 공적 신뢰감이 있어야 한다.지나치게 가벼운 스타트업형 UI나 광고 랜딩페이지 느낌을 피한다.
둘째, 정책 SaaS형 명료성이 있어야 한다.질문, 카드, 브리핑, 근거, 검수 정보가 체계적으로 보여야 한다.
셋째, 생활 밀착형 친절함이 있어야 한다.딱딱한 관공서 UI처럼 보이기보다, 시민이 편하게 읽을 수 있어야 한다.
넷째, 정치적 과잉 연출을 자제해야 한다.깃발, 과도한 슬로건, 전투적 그래픽, 강한 배경 이미지 남용은 줄이고, 정보 전달 중심의 구조를 유지한다.
2.3 시각적 비유
디자인적으로 hanjunho.ai는 다음 세 가지 사이의 균형을 목표로 한다.
공공 데이터 포털정책 브리핑 SaaS검증된 매거진형 해설 사이트
즉, 선거 포스터 사이트도 아니고, 일반 뉴스 미디어도 아니며, 단순한 블로그도 아니다.정책과 이슈를 질문형으로 구조화한 신뢰 가능한 인터페이스가 되어야 한다.
3. 브랜드 시각 키워드
hanjunho.ai의 핵심 시각 키워드는 다음과 같다.
명료함신뢰정돈됨설명 가능성생활 연결성검증 가능성
이 키워드는 모든 페이지와 컴포넌트의 최상위 판단 기준이 된다.어떤 UI 요소가 추가될 때 이 여섯 키워드와 맞지 않으면 제거 또는 수정한다.
4. 컬러 시스템
4.1 기본 원칙
컬러는 적게 쓰고, 역할이 분명해야 한다.hanjunho.ai는 색으로 시선을 끄는 사이트가 아니라, 정보 위계를 분명히 하는 사이트다.
색상은 크게 다섯 그룹으로 운용한다.
첫째, Primary Blue 계열둘째, Support Blue/Gray 계열셋째, Trust Gold 계열넷째, Neutral Background 계열다섯째, Semantic Alert/Success 계열

#004EA2
역할: 브랜드 대표색, 주요 버튼, 링크 강조, 핵심 제목 강조, 질문형 정책 플랫폼의 정체성 표현
사용 위치:상단 GNB 로고/브랜드명주요 CTA 버튼Answer Card 핵심 제목 강조페이지 섹션 제목의 보조 강조링크 hover 강조
주의:배경 전체를 파란색으로 덮지 않는다.Trust Blue는 “강조” 색이지, “면적” 색이 아니다.

5. 타이포그래피 시스템
5.1 기본 원칙
타이포그래피는 hanjunho.ai의 핵심 경쟁력이다.이 사이트는 글을 많이 읽는 사이트이므로, 폰트 선택보다 위계, 행간, 문단 리듬이 중요하다.
기본 원칙은 다음과 같다.
문장은 짧게 끊되 너무 파편화하지 않는다.행간은 넉넉하게 유지한다.제목은 강하고 짧게, 본문은 부드럽고 명확하게 간다.모바일 기준 가독성을 최우선으로 한다.
5.2 권장 폰트
기본 한글 폰트는 Pretendard를 권장한다.이유는 다음과 같다.
가독성이 좋고웹플로우 환경에서 안정적이며정책 플랫폼에 어울리는 현대적이면서도 과하지 않은 인상을 주기 때문이다.
보조 시스템 폰트는 sans-serif로 둔다.
5.3 폰트 위계
Display / Hero H1
크기: 48~60px굵기: Bold행간: 120~130%
사용 위치:홈 H1Answer Card 상세 H1이슈브리핑 상세 H1프로필 페이지 H1
Section Title
크기: 34~42px굵기: Bold행간: 125~135%
사용 위치:섹션 제목카드 블록 주요 제목정책 페이지 중단 주요 구간
Card Title / Subhead
크기: 24~30px굵기: Bold행간: 130~140%
사용 위치:카드 제목브리핑 카드 제목프로필 섹션 소제목
Body Large
크기: 18~20px굵기: Regular 또는 Medium행간: 160%
사용 위치:히어로 서브카피요약 박스 본문해설 본문 상단
Body Base
크기: 16px굵기: Regular행간: 160%
사용 위치:일반 본문설명문근거 자료 설명폼 라벨
Meta / Caption
크기: 13~14px굵기: Medium행간: 150%
사용 위치:업데이트일매체명카테고리작성/검수 라벨
5.4 강조 규칙
한 문단 안에서 굵게 처리하는 표현은 1~2개만 허용한다.링크는 파란색 + 밑줄 hover로 명확히 보여준다.긴 문장을 모두 bold 처리하는 것은 금지한다.
6. 레이아웃 시스템
6.1 전체 콘텐츠 폭
데스크톱 기준 최대 콘텐츠 폭은 1200px ~ 1280px 수준을 권장한다.너무 넓으면 긴 문서형 페이지에서 가독성이 떨어진다.
6.2 그리드 구조
데스크톱 기준 12-column grid를 기본으로 한다.태블릿은 8-column, 모바일은 4-column 또는 단일 컬럼 구조로 전환한다.
6.3 여백 시스템
가장 중요한 것은 일정한 vertical rhythm이다.
권장 spacing scale은 다음처럼 운영한다.
4px8px12px16px24px32px48px64px96px
페이지 섹션 간격은 64px 또는 96px를 기본으로 한다.카드 내부 패딩은 20~28px 수준이 적당하다.작은 칩과 버튼 내부 패딩은 10~16px 수준을 사용한다.
6.4 컨테이너 규칙
각 섹션은 가능한 한 하나의 큰 container 안에서 정렬한다.섹션마다 정렬 기준이 달라지면 사이트가 템플릿 조각처럼 보인다.정렬 기준은 기본적으로 좌정렬을 유지한다.
7. 모서리, 테두리, 그림자 규칙
7.1 라운드
hanjunho.ai는 너무 날카로운 UI도, 너무 둥근 UI도 피해야 한다.권장 반경은 다음과 같다.
칩: 14~16px버튼: 14~18px카드: 20~24px대형 컨테이너: 24~28px
7.2 테두리
카드와 컨테이너는 아주 얇은 border를 기본으로 사용한다.border color는 #D8E2ED 계열을 기본으로 한다.
7.3 그림자
그림자는 강하지 않게, 매우 은은하게 쓴다.정치 플랫폼에서 과한 드롭섀도는 저급하게 보일 수 있다.
권장 기본 그림자 예시:0 8px 24px rgba(16, 37, 62, 0.06)
검수 배지나 강조 카드에는 그림자보다 배경색 차이를 우선 사용한다.
8. 아이콘·이미지 시스템
8.1 아이콘 원칙
아이콘은 장식보다 의미 전달용으로 사용한다.너무 많은 아이콘 스타일을 섞지 않는다.선형(line) 아이콘 위주가 적절하다.
권장 아이콘 사용 위치:검색근거 자료업데이트일브레드크럼작성/검수투표 안내질문·제안·제보
8.2 이미지 원칙
히어로 이미지는 감정과 인물 인지를 담당한다.카드 이미지 남용은 금지한다.정책 카드와 브리핑은 이미지보다 텍스트 구조가 더 중요하다.
프로필 이미지는 실제 인물 중심의 신뢰감 있는 컷을 사용한다.placeholder나 템플릿 기본 이미지, 너무 연출된 스톡 이미지 사용은 금지한다.
9. 핵심 컴포넌트 가이드
9.1 GNB(상단 헤더)
구성:로고/브랜드명전역 메뉴 7개모바일 햄버거
디자인 원칙:화이트 배경하단 얇은 구분선현재 페이지 강조는 색상과 밑줄 또는 굵기로 처리hover는 파란색 또는 진한 네이비 전환
금지:과도한 배경색, 그림자 과다, 메뉴 수직정렬 흔들림
9.2 히어로 섹션
역할:페이지 정체성과 핵심 행동을 첫 화면에서 설명
구성:소형 라벨강한 H1설명문대표 CTA 2~3개필요 시 검색창 또는 신뢰 배지
디자인 원칙:왼쪽 정렬문장 길이 짧게핵심 행동 버튼은 1개만 Primary나머지는 Secondary 또는 Ghost 스타일
9.3 검색창 컴포넌트
역할:Ask AI 페이지의 핵심 인터랙션
구성:입력 필드플레이스홀더검색 버튼보조 힌트
디자인 원칙:높이 충분히 크게모바일에서도 오타 없이 누르기 쉬운 크기포커스 시 Light Blue 또는 Blue outlineplaceholder는 질문 예시형 문장 사용
예시 플레이스홀더:GTX-R, 전세사기 대책, 청년 일자리, 1기 신도시처럼 자연어로 찾아보세요
9.4 칩(Chip) 컴포넌트
칩은 hanjunho.ai에서 매우 중요하다.카테고리, 도메인, 검수, 매체, Verdict 같은 정보를 짧게 요약한다.
칩 종류는 다음처럼 구분한다.
정책 도메인 칩카테고리 칩검수 칩이슈브리핑 라벨 칩팩트체크 Verdict 칩
디자인 원칙:작고 정돈된 pill 형태색상은 역할 기반으로만 다르게한 화면에 칩이 너무 많아지지 않게 제한
검수 칩 예시:배경 #FFF7E8텍스트 #8A6200
9.5 버튼 컴포넌트
버튼은 세 종류로만 운영하는 것이 좋다.
Primary Button
배경: Trust Blue텍스트: White용도: 핵심 CTA
Secondary Button
배경: White테두리: Trust Blue텍스트: Trust Blue용도: 보조 CTA
Tertiary / Text Button
배경 없음텍스트만용도: 보조 링크, 관련 콘텐츠, 원문 기사 보기
버튼 문구 원칙:동사 중심짧고 분명하게한 버튼에 한 행동만
예시:정답 보기관련 정책 보기원문 기사 보기프로필 보기투표 안내 보기
9.6 Answer Card 리스트 카드
역할:정책 탐색과 스캔을 빠르게 만드는 핵심 카드
권장 구조:Policy Domain 칩Category 칩검수 배지카드 제목User Query 1줄Snippet 2줄업데이트일CTA
디자인 원칙:질문이 보이되 제목과 충돌하지 않게본문은 2줄 내외카드 높이는 일정하게모바일에서 1열, 데스크톱에서 2~3열
현재 사이트처럼 너무 많은 카드가 한 화면에 펼쳐지고, 오분류 카드와 샘플 카드가 함께 보이면 탐색성이 크게 떨어진다. 따라서 카드 수보다 카드 품질과 위계를 우선한다.
9.7 Answer Card 상세 상단 구조
역할:정답 페이지임을 첫 화면에서 보여주는 핵심 영역
구성:breadcrumbPolicy Domain / Category / 검수 / 업데이트일H1질문 말풍선명쾌한 정답 박스
디자인 원칙:스크롤 전 첫 화면 안에질문과 3문장 정답이 보이도록 구성메타 칩과 H1의 간격은 좁고질문과 정답 박스의 간격은 충분히 둔다
정답 박스는 Light Blue 배경을 사용하고,질문 박스는 Soft Gray 배경을 써서 시각적으로 역할을 구분한다.
9.8 실행 방안 카드
역할:정책을 추상적 슬로건이 아니라 실행 구조로 보이게 만드는 카드
구성:번호 또는 아이콘Action TitleAction Body
디자인 원칙:3개를 나란히 놓되 모바일에서는 세로 스택각 카드의 높이는 가급적 맞춘다제목은 Bold본문은 2~4줄 정도가 적당하다
9.9 근거 자료 리스트
역할:신뢰 확보
구성:자료명출처기관 또는 설명열기 버튼
디자인 원칙:링크를 텍스트만으로 두지 않고,리스트형 row로 보여준다.행마다 hover 강조가 있으면 좋다.
9.10 이슈브리핑 카드
역할:기사형 해설 허브에서 이슈를 스캔하게 하는 카드
구성:브리핑 라벨Issue TopicPublisher날짜제목2~3문장 요약관련 정책 보기 CTA
디자인 원칙:뉴스 카드처럼 너무 빽빽하게 만들지 않는다.“기사 링크”가 아니라 “해설 페이지”라는 느낌이 나야 한다.매체명과 날짜는 상단 메타로 분명히 보이게 한다.
9.11 이슈브리핑 상세 상단 구조
역할:기사 출처와 핵심 쟁점을 빠르게 전달
구성:breadcrumb이슈브리핑 라벨Issue TopicPublisher / 날짜 / 업데이트일H1Summary 박스다룬 기사 블록핵심 쟁점 3가지
디자인 원칙:기사 출처 블록과 해설 본문은 시각적으로 분리Summary 박스는 Light Red 또는 Alert Surface 계열로 변주 가능핵심 쟁점 카드는 단순하며 반복 리듬이 있어야 한다
9.12 프로필 카드
역할:팩트 인증센터와 상세페이지 하단에서 인물의 역할을 짧게 설명
구성:프로필 이미지이름역할짧은 소개프로필 보기 CTA
디자인 원칙:이름보다 역할 정의가 분명해야 한다프로필 이미지는 실제 사진을 사용과도한 장식 없이 정돈된 카드형
9.13 프로필 상세 히어로
역할:“왜 이 사람이 중요한가”를 첫 화면에서 설명
한준호 페이지는공식 발화자 라벨강한 H1설명문권위 배지를 사용한다.
김태영 교수 페이지는총괄 검수자 라벨강한 H1설명문권위 배지를 사용한다.
디자인 원칙:두 페이지는 공통 프레임을 유지하되,한준호는 Blue 중심,김태영은 Gold/Neutral 보조 강조를 더해 역할 차이를 드러낸다.
9.14 팩트 인증센터 신뢰 블록
역할:누가 답하고 누가 검수하는지 구조를 보여줌
구성:한준호김태영 교수캠프 AI 정책본부3개 카드 + 역할 설명
디자인 원칙:사람 카드 나열이 아니라,“질문 → 작성 → 검수 → 공개” 흐름과 함께 보여주는 것이 좋다.
9.15 폼 컴포넌트
적용 페이지:질문·제안·제보 페이지문의 성격의 보조 페이지
구성:라벨입력 필드유형 선택상세 내용제출 버튼안내문
디자인 원칙:관공서 폼처럼 딱딱하지 않게하지만 너무 캐주얼하지도 않게필수값 표시와 오류 메시지는 분명하게
오류 메시지 색상은 Alert Red 계열완료 메시지는 Success Green 계열을 사용한다.
9.16 빈 상태(Empty State) 컴포넌트
hanjunho.ai는 필터와 검색이 많으므로 empty state가 중요하다.
구성:짧은 설명문재탐색 CTA
예시:조건에 맞는 콘텐츠가 아직 없습니다.다른 질문이나 카테고리로 다시 찾아보세요.
디자인 원칙:허전하지 않되 과하지 않게도와주는 톤으로 작성

11. 모바일 반응형 가이드
11.1 기본 원칙
모바일에서 정보량을 줄이는 것이 아니라, 정보 위계를 더 명확히 해야 한다.핵심은 숨기는 것이 아니라 순서를 조정하는 것이다.
11.2 모바일 우선순위
Answer Card 상세에서는H1보다 질문과 Snippet이 먼저 눈에 들어오게 한다.
이슈브리핑 상세에서는제목 다음에 Summary와 기사 출처를 바로 배치한다.
프로필 페이지에서는권위 배지보다 역할 정의 문장을 우선 보이게 한다.
11.3 모바일 레이아웃 규칙
카드 3열 금지모바일은 기본 1열칩은 2줄 이상 넘어가면 정리버튼은 터치 가능한 높이 확보말풍선과 요약 박스는 가로 여백 좁히되 내부 패딩 유지
12. 접근성 가이드
정치·정책 사이트는 정보 접근성이 매우 중요하다.따라서 최소한 다음 원칙을 지킨다.
본문 텍스트와 배경 대비를 충분히 확보한다.버튼은 색만으로 상태를 구분하지 않는다.칩과 배지는 텍스트를 포함한다.링크는 hover 외에도 시각적으로 링크임이 드러나야 한다.헤딩 레벨은 페이지 구조에 맞게 사용한다.폼에는 라벨을 생략하지 않는다.
13. Webflow 구현 가이드
13.1 클래스 네이밍 원칙
Webflow에서 지나치게 세부적으로 클래스가 분기되면 유지보수가 어렵다.따라서 다음 같은 네이밍 패턴을 권장한다.
section-herosection-card-listsection-briefing-listsection-profilecomponent-chipcomponent-button-primarycomponent-button-secondarycomponent-card-answercomponent-card-briefingcomponent-card-profilecomponent-breadcrumbcomponent-metadata-rowcomponent-evidence-list
즉 “페이지명 기준”보다 “컴포넌트 기준”으로 클래스 구조를 잡는 편이 좋다.
13.2 Combo Class 사용 원칙
기본 컴포넌트 + 변형 구조를 쓴다.
예시:component-cardcomponent-card answercomponent-card briefingcomponent-card profile
이렇게 하면 유지보수가 쉬워진다.
13.3 CMS 컴포넌트 연결 원칙
디자인 컴포넌트는 최대한 고정하고,CMS는 텍스트/이미지/링크만 채워 넣는 방식으로 간다.디자인과 데이터 로직을 한 컴포넌트 안에서 과도하게 섞지 않는다.
14. 금지 패턴
다음 패턴은 사용하지 않는다.
너무 진한 풀컬러 배경이 반복되는 페이지텍스트보다 이미지가 더 큰 카드강한 네온/형광 계열 색상카피보다 장식 그래픽이 더 눈에 띄는 섹션슬라이더를 남발하는 구조굵은 그림자가 남용된 카드하나의 섹션 안에서 정렬 기준이 2개 이상 섞이는 배치Gallery/Contact를 상위 정보구조처럼 보이게 하는 시각 처리기업형 SaaS 랜딩페이지처럼 보이는 영문/비즈니스 카피 배치
15. 시각적 완성도 체크 기준
최종 산출물이 완성도 있다고 판단하는 기준은 다음과 같다.
홈이 질문을 시작하게 만드는가클리어맵이 정책을 탐색하게 만드는가Answer Card는 정답 페이지처럼 읽히는가이슈브리핑은 기사형 해설 페이지처럼 읽히는가팩트 인증센터는 신뢰 허브처럼 읽히는가한준호 페이지는 공식 발화자 페이지처럼 읽히는가김태영 페이지는 총괄 검수자 권위 페이지처럼 읽히는가모든 페이지에서 Blue, Gold, Red의 역할이 일관되는가카드와 섹션 간 여백 리듬이 일정한가모바일에서도 정보 위계가 무너지지 않는가
16. 최종 요약
이 가이드의 핵심은 간단하다.
hanjunho.ai는 선거용 홍보 페이지처럼 보여서는 안 된다.이 사이트는 질문에 답하고, 이슈를 해설하고, 신뢰를 설명하는 정책 플랫폼처럼 보여야 한다.
그러기 위해서는색상은 역할 기반으로 쓰고,타이포그래피는 가독성과 위계에 집중하고,카드는 정보 스캔을 돕고,상세페이지는 첫 화면에 핵심 구조를 보여주며,프로필과 검수 정보는 신뢰를 강화하는 방향으로 설계되어야 한다.
이 문서는 이후QA/UAT 체크리스트와런치·운영 플레이북작성 및 실제 구현 검수의 시각 기준으로 함께 사용한다.
